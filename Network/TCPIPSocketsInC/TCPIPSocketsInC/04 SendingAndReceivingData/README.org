* Sending And Receiving Data

** 编码整数
小端顺序：从最低有效位，即数字的“右”端开始编码整数。编码方向和地址增长方向一致
大端顺序：从最高有效位，即数字的“左”端开始编码整数。编码方向和地址增长方向相反
Tips：
（1）在Sockets API中，地址和端口总是使用网络字节顺序。
1 0000 = 2^4
  1111 = -1  = tobit(2^4-1)
  1110 = -2  = tobit(2^4-2)
  1101 = -3  = tobit(2^4-3)
  1100 = -4  = tobit(2^4-4)
  1011 = -5  = tobit(2^4-5)
  1010 = -6  = tobit(2^4-6)
  1001 = -7  = tobit(2^4-7)
  1000 = -8  = tobit(2^4-8) = -2^(4-1)
